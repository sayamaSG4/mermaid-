# GitHub 入門ガイド（Git 経験者向け）

Git の基本操作は理解しているものの、GitHub はこれから使い始める方向けに、
アカウント作成から代表的な画面や操作までをまとめました。ブラウザでの操作を中心に、
Git リポジトリとのつながりを意識できるように解説しています。

---

## 1. GitHub を使う準備

### 1-1. アカウント作成
1. [GitHub](https://github.com/) にアクセスし、右上の **Sign up** をクリックします。
2. メールアドレス・パスワード・ユーザー名を入力し、アカウントを作成します。
3. 送られてくる確認メールで認証すれば完了です。

### 1-2. プロフィールの設定
- 右上のプロフィールアイコン → **Your profile** から、アイコン画像や自己紹介文を設定できます。
- `Settings` → `Profile` で表示名や会社名なども変更できます。

---

## 2. GitHub の基本用語と画面

| 用語 | 説明 | 主な画面への行き方 |
|------|------|--------------------|
| Repository（リポジトリ） | プロジェクトの保管場所。Git のリモートリポジトリ。 | 右上の `＋` → **New repository**、またはプロフィールページの **Repositories** タブ |
| Branch（ブランチ） | 作業用の枝。Git のブランチと同じ概念。 | リポジトリ内上部のブランチセレクター（`main` 表示部分） |
| Commit | 変更履歴の単位。Git と同じ。 | リポジトリ内の **Commits** タブや Pull Request 画面 |
| Pull Request（PR） | ブランチでの変更を main へ取り込むためのレビュー申請。 | リポジトリ上部の **Pull requests** タブ |
| Issues | 課題・要望を管理するチケット。 | リポジトリ上部の **Issues** タブ |

### 主な画面の見方
- **Repository トップ**: ファイル一覧、README、最新コミットなどが表示されます。
- **Code タブ**: ファイルの閲覧や編集、ブランチ切り替え、クローン URL の取得ができます。
- **Pull requests タブ**: 既存 PR の一覧と作成ボタンが並びます。
- **Actions タブ**: CI（自動テストやビルド）の状況が確認できます。
- **Settings タブ**: リポジトリの公開範囲、ブランチ保護、メンバー管理などを設定します。

---

## 3. 新しいリポジトリを作成する
1. 右上の `＋` メニューから **New repository** を選びます。
2. **Repository name** にリポジトリ名を入力し、公開範囲（Public/Private）を選びます。
3. 必要に応じて README、`.gitignore`、ライセンスを初期生成します。
4. **Create repository** をクリックすると空のリポジトリができます。
5. 既存のローカルリポジトリを接続する場合は、表示される手順（`git remote add origin ...`）に従って push します。

---

## 4. ブランチを作成する

### GitHub（ブラウザ）でブランチを作る
1. リポジトリの **Code** タブで、左上のブランチセレクター（通常 `main` と表示）をクリックします。
2. 検索欄に新しいブランチ名を入力し、表示される `Create branch: <branch-name>` を選択します。
3. 作成したブランチに切り替わった状態でファイル編集や PR 作成ができます。

### ローカルからブランチを push して作る
```bash
git checkout -b feature/update-guide
git push -u origin feature/update-guide
```
- GitHub 上でリポジトリを開くと、新しいブランチがブランチセレクターに表示されます。

---

## 5. ファイル・フォルダーを作成する

### ブラウザで新しいファイルを追加する
1. **Add file** → **Create new file** をクリック。
2. `Name your file...` に `docs/guide.md` のように入力すると、`docs` フォルダーが同時に作成されます。
3. 編集エリアに内容を入力し、下部の **Commit changes** でコミットします。
   - `Commit message` は変更内容が分かる短い文章を入力。
   - 直接 `main` にコミットするか、新しいブランチを切って PR を作成するか選べます。

### ブラウザでフォルダーのみを作成する
- GitHub 上ではフォルダー単体は作れません。`folder/.gitkeep` のように空ファイルを追加してフォルダーを表現します。

### ローカルで作成して push する
```bash
mkdir -p docs/images
touch docs/images/.gitkeep
git add docs
git commit -m "Add docs folder"
git push origin <branch-name>
```

---

## 6. Pull Request を作成する流れ
1. 作業ブランチを GitHub に push すると、リポジトリトップに **Compare & pull request** ボタンが表示されます。
2. ボタンを押して PR 作成画面へ進み、タイトルと説明を記入します。
3. 変更内容を確認し、必要ならレビュアーやラベル、プロジェクトを設定します。
4. **Create pull request** をクリックしてレビューを依頼します。
5. レビューで指摘されたら修正コミットを追加し、`git push` すれば PR に反映されます。
6. 承認後は **Merge pull request** → **Confirm merge** で main に取り込みます。

---

## 7. Issue で課題管理を始める
1. リポジトリの **Issues** タブ → **New issue** をクリック。
2. テンプレートがある場合は選択し、タイトルと内容を記載します。
3. 担当者（Assignees）、ラベル（Labels）、期限（Milestone）などを設定します。
4. 作業が完了したら issue を **Close** して終了です。

---

## 8. 通知とコラボレーション
- 右上のベルアイコンから、PR や issue の更新をまとめて確認できます。
- リポジトリ右上の **Watch** ボタンで通知の強さを調整できます。
- コメント内で `@ユーザー名` と書くとメンションできます。
- `README` や `Wiki` でプロジェクトの説明を整えると、チーム内共有がスムーズになります。

---

## 9. さらに学ぶために
- GitHub の公式ドキュメント: <https://docs.github.com/>
- GitHub Skills（ブラウザ上で学べるチュートリアル）: <https://skills.github.com/>
- Git コマンドと合わせて、Pull Request ベースの開発フローを小さなプロジェクトで試すと理解が深まります。

---

### まとめ
- GitHub は Git のリモートホスティングと、コラボレーションを支える機能がセットになったサービスです。
- ブランチと PR を活用することで、安全にレビューを受けながら変更を共有できます。
- まずは小さなリポジトリを作成し、ブラウザ上でファイルを追加・編集するところから始めてみましょう。
